### ✨깃 컨벤션

- commit 규칙
    - init: 최초 커밋: Unity 프로젝트 생성
    - feat: 기능 추가
    - refactor: 기능 개선
    - add: 에셋, .cs 등 파일 추가
    - move: 파일 이동, 코드 이동 등
    - remove: 파일 삭제
    - art: UI 개선
    - fix: 버그 수정
    - chore: 기타 잡일
    - docs: 리드미 등 문서 수정
 
- branch
    - dev (하루에 한 번 main 업데이트)
    - branch는 기능마다 1개씩.
        - 기능 추가 시: feat/(기능 이름)
        - UI 추가 시: UI/(추가하는 UI) ->맵처럼 큰 규모만.
